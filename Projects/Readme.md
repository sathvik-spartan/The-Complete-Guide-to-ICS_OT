# Top 50+ Projects for Learning OT Cybersecurity

1. **Hack Your First PLC:** Use a free virtual PLC and walk through learning how easy it is to compromise OT/ICS assets.

2. **Use GenAI to Create Your Own OT Tools:** As you learn, think of your own OT/ICS security tools to create!

3. **Build a Virtual PLC:** Install OpenPLC on a Linux VM to build your own first virtual PLC.

4. **Use Shodan:** Access shodan.io to look for OT/ICS assets that are exposed to the public Internet and see what you can find!

5. **Spin up Labshock:** Labshock has two great free simulated environments for learning from oil fields to water treatment.

6. **Check Out OT Tools in Kali Linux:** Explore which OT-specific tools exist within Kali. Something doesn't exist? Create it!

7. **Integrate Gemini GenAI with Kali Linux:** Deploy the `gemini-cli` utility to experiment with automating security testing.

8. **Create OT Wireshark Filters:** Make OT filters, such as capturing Modbus read requests or finding S7 commands.

9. **Write an OT Cybersecurity Policy:** While not fun for most to create, this can speak volumes to hiring managers.

10. **Create an SBOM:** Build out a Software Bill of Materials for an OT/ICS asset and supporting OT/ICS cybersecurity.

11. **Review an SBOM for Vulnerabilities:** Use tools to review an asset's software components for relevant vulnerabilities.

12. **Build an OT Tabletop:** Make sure to include a scenario with real-world injects for a specific type of environment.

13. **Create a LinkedIn Post:** Take a topic you know in OT/ICS cybersecurity and create a LinkedIn post to share with others!

14. **Map Real Incident to MITRE ATT&CK for ICS:** Map incidents and their aspects to the ATT&CK platform for learning.

15. **Create a Lessons Learned List:** Review a real-world incident that occurred and create a list of action items to prevent it.

16. **Incident Response Planning:** Outline an IR plan for your particular environment - whether you work in it or want to.

17. **Risk Assessment Walkthrough:** Provide a walkthrough of how an ISA/IEC 62443 risk assessment works in OT/ICS.

18. **Physical Security Checklist:** Create a checklist of areas to audit when conducting a physical security assessment in OT.

19. **Applying ISA/IEC 62443:** Provide a checklist of controls to deploy for a section of a network based on 62443 SL-1.

20. **Learn to Program a PLC:** Take an hour a day to learn how to program a PLC so you can be comfortable with the basics.

21. **Deploy Network Monitoring with Security Onion:** Use the open source platform and its OT protocol support for testing.

22. **Create an OT Network Diagram:** Document a simple OT network including Purdue levels, VLANs, firewalls, switches, HMIs, PLCs, and historians.

23. **Examine Dangers of Unauthenticated Protocols:** Spend time learning why protocols like Modbus are critical vulns!

24. **Configure an OT Honeypot:** Use Conpot, T-pot, or create your own for testing. And expose it to the Internet if you can!

25. **Launch Password Attacks Against OT Systems:** Use tools to automate password guessing attacks to gain access.

26. **Bootable USB for Response:** Create a bootable USB to include open source tools needed for incident response in OT.

27. **Create a Tool Cheatsheet:** Help others learn by creating a quickstart guide on how to use a specific tool like Shodan.

28. **Launch GRFICSv3 for a Physical Walkthrough:** Walk through a virtual site to find security issues without traveling.

29. **Create Your 1st HMI:** Start out by visualizing what's happening with a PLC using open source like ScadaBR/FUXA.

30. **Try out GRASSMARLIN:** An oldie, but goodie, but unfortunately hasn't been supported for a LONG time.

31. **Or Try an Alternative:** Folks have been working on alternatives to GRASSMARLIN to check out like Gridwolf.

32. **Harden an EWS:** Show how you would harden an OT engineering workstation and document the steps to show.

33. **Don't Forget to Backup:** Establish a backup & recovery program along with test backups and restores for assurance.

34. **Nmap Scans in OT:** Conduct different types of Nmap scans in your OT test environment to learn how scans work.

35. **Explore Nmap Alternatives:** Full active scanning isn't always necessary; look at different "active polling" options.

36. **Review Known Vulnerabilities:** Review the CISA database for known vulnerabilities on OT assets and evaluate for risk.

37. **Use the ICS Advisory Project:** Want more on vulns? Check out Dan Ricci's project for more metrics and greater depth.

38. **Read the Dragos Year in Review Report:** Check out this free, vast treasure trove of info on the current landscape.

39. **Use Vendor Research to Build an Attack Tool:** Review research to emulate an adversary's tools like FrostyGoop.

40. **Use Vendor Research to Build a Defense Tool:** Build a control that protects against a known OT security incident.

41. **Create Malicious PLC Logic:** Inject malicious code into PLC logic that would have a significant impact in a test network.

42. **Use Volatility on an EWS:** What type of interesting information could you find in the memory of a workstation?

43. **Set Up an OT SIEM:** Use an open source SIEM platform to configure a test platform for correlations and investigations.

44. **Prepare a Threat Hunt Plan:** Take a well-known OT cyber incident and create steps for finding the same activity.

45. **Explore OT Protocols:** Take the time to examine how common OT protocols like Modbus, DNP3, S7 actually work.

46. **Create Your Own Virtual Lab:** Pick a type of environment and use AI to create a simulated process for testing against.

47. **Deploy a Historian for Telemetry:** Set up a system for gathering telemetry data from assets in the OT network.

48. **Build a Raspberry Pi PLC:** Want a low-cost solution to build your own physical PLC? Try one of the Raspberry Pi kits!

49. **Perform an OT Firewall ACL Review:** Set up a firewall and configure firewall rules to inspect using least privilege.

50. **Capture and Analyze OT Traffic:** Record network traffic from a simulated environment and identify normal communications between devices.

51. **Deploy an ICS IDS Rule:** Write or modify a Suricata or Zeek rule to detect suspicious OT protocol activity.

52. **Build an Asset Inventory:** Create an OT asset inventory including devices, firmware versions, network addresses, and criticality.

53. **Baseline Normal Network Traffic:** Establish what "normal" OT communications look like and document expected device behavior.

54. **Create an OT Network Segmentation Plan:** Design zones and conduits for an industrial network following ISA/IEC 62443 principles.

55. **Decode OT Protocol Packets:** Manually inspect Modbus, DNP3, or S7 packets and identify function codes and data fields.

56. **Create a Short Video:** Share your knowledge with others that don't know what you do. Share it to your network or Open Source it!
